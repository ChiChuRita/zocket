# Strategy Notes

This folder is the canonical home for Zocket's product, strategy, and architecture notes.

Naming convention:

- lowercase kebab-case
- one note per topic
- direct titles that state the document's job

Current notes:

- [`context.md`](./context.md) — the shortest description of what Zocket is, who it is for, and how the product works.
- [`pmf.md`](./pmf.md) — the current product-market-fit thesis, wedge, and adoption signals.
- [`competition.md`](./competition.md) — the competitor frame and the positioning choices it implies.
- [`features.md`](./features.md) — the feature backlog and the product bets worth building next.
- [`infra.md`](./infra.md) — the infrastructure direction, split between v1 and the longer-term platform.

V1 notes:

- [`v1/auth.md`](./v1/auth.md) — tenant identity, auth, routing, and isolation for the MVP.
- [`v1/aws.md`](./v1/aws.md) — the concrete AWS setup for the MVP infrastructure.
- [`v1/platform.md`](./v1/platform.md) — the MVP account, dashboard, and CLI deploy experience.

Rule of thumb:

- if it explains what the product is or should become, it belongs here
- if it is public docs, generated output, or legacy reference material, it does not
