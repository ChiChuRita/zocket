# Middleware

Middleware functions run before handlers and can add data to the context, perform validation, or implement cross-cutting concerns like authentication and logging.

## Basic Middleware

Create middleware with `zo.message.use()`:

```typescript
const authenticated = zo.message.use(({ ctx, payload }) => {
  if (!ctx.isAuthenticated) {
    throw new Error("Unauthorized");
  }

  return {
    verified: true,
  };
});
```

Apply middleware to messages:

```typescript
const router = {
  admin: {
    action: authenticated.incoming({
      payload: z.object({ command: z.string() }),
    }),
  },
};
```

## Middleware Function

Middleware receives `ctx` and `payload`:

```typescript
zo.message.use(({ ctx, payload }) => {
  // ctx: Current context (user data, clientId, etc.)
  // payload: Message payload (type depends on message)

  // Return object to add to context
  return {
    timestamp: Date.now(),
    processed: true,
  };
});
```

## Adding to Context

Data returned from middleware is added to the context:

```typescript
const withTimestamp = zo.message.use(() => {
  return {
    timestamp: Date.now(),
    requestId: crypto.randomUUID(),
  };
});

const router = {
  api: {
    call: withTimestamp.incoming({
      payload: z.object({ data: z.string() }),
    }),
  },
};

const appRouter = zo.router(router, {
  api: {
    call: ({ payload, ctx }) => {
      ctx.timestamp; // number (from middleware)
      ctx.requestId; // string (from middleware)
      ctx.userId; // string (from onConnect)
    },
  },
});
```

## Authentication Middleware

Common pattern for protecting routes:

```typescript
const requireAuth = zo.message.use(({ ctx }) => {
  if (!ctx.userId) {
    throw new Error('Must be logged in');
  }

  return {
    authenticated: true,
  };
});

const requireAdmin = zo.message.use(({ ctx }) => {
  if (ctx.role !== 'admin') {
    throw new Error('Must be admin');
  }

  return {
    isAdmin: true,
  };
});

const router = {
  user: {
    profile: requireAuth.incoming({...}),
  },
  admin: {
    deleteUser: requireAdmin.incoming({...}),
  },
};
```

## Role-Based Middleware

Check user roles:

```typescript
const requireRole = (role: string) => {
  return zo.message.use(({ ctx }) => {
    if (ctx.role !== role && ctx.role !== 'admin') {
      throw new Error(`Requires ${role} role`);
    }

    return {
      authorizedRole: role,
    };
  });
};

const router = {
  moderator: {
    ban: requireRole('moderator').incoming({...}),
    mute: requireRole('moderator').incoming({...}),
  },
};
```

## Permission-Based Middleware

Check specific permissions:

```typescript
const requirePermission = (permission: string) => {
  return zo.message.use(async ({ ctx }) => {
    const hasPermission = await checkUserPermission(ctx.userId, permission);

    if (!hasPermission) {
      throw new Error(`Requires permission: ${permission}`);
    }

    return {
      permission,
      granted: true,
    };
  });
};

const router = {
  posts: {
    create: requirePermission('posts.create').incoming({...}),
    delete: requirePermission('posts.delete').incoming({...}),
  },
};
```

## Composing Middleware

Chain multiple middleware:

```typescript
const logged = zo.message.use(({ ctx, payload }) => {
  console.log(`[${ctx.userId}] called with`, payload);
  return { logged: true };
});

const timed = zo.message.use(() => {
  return { startTime: Date.now() };
});

const protectedLogged = logged.use(({ ctx }) => {
  if (!ctx.userId) {
    throw new Error('Unauthorized');
  }
  return { protected: true };
});

const fullyProtected = protectedLogged.use(timed);

const router = {
  api: {
    sensitiveAction: fullyProtected.incoming({...}),
  },
};
```

## Async Middleware

Middleware can be async:

```typescript
const validateApiKey = zo.message.use(async ({ ctx, payload }) => {
  const isValid = await checkApiKey(payload.apiKey);

  if (!isValid) {
    throw new Error("Invalid API key");
  }

  return {
    apiKeyValid: true,
  };
});
```

## Rate Limiting Middleware

Implement rate limiting:

```typescript
const rateLimits = new Map<string, number[]>();

const rateLimit = (maxRequests: number, windowMs: number) => {
  return zo.message.use(({ ctx }) => {
    const now = Date.now();
    const userRequests = rateLimits.get(ctx.clientId) || [];

    const recentRequests = userRequests.filter(
      time => now - time < windowMs
    );

    if (recentRequests.length >= maxRequests) {
      throw new Error('Rate limit exceeded');
    }

    recentRequests.push(now);
    rateLimits.set(ctx.clientId, recentRequests);

    return {
      rateLimitRemaining: maxRequests - recentRequests.length,
    };
  });
};

const router = {
  api: {
    call: rateLimit(10, 60000).incoming({...}), // 10 per minute
  },
};
```

## Validation Middleware

Additional validation beyond schema:

```typescript
const validateBusinessRules = zo.message.use(({ ctx, payload }) => {
  if (payload.amount > ctx.balance) {
    throw new Error("Insufficient balance");
  }

  if (payload.recipient === ctx.userId) {
    throw new Error("Cannot send to self");
  }

  return {
    validated: true,
  };
});

const router = {
  payment: {
    send: validateBusinessRules.incoming({
      payload: z.object({
        amount: z.number(),
        recipient: z.string(),
      }),
    }),
  },
};
```

## Logging Middleware

Log requests:

```typescript
const withLogging = zo.message.use(({ ctx, payload }) => {
  console.log(`[${new Date().toISOString()}] ${ctx.userId}:`, payload);

  return {
    loggedAt: Date.now(),
  };
});

const router = {
  api: {
    action: withLogging.incoming({...}),
  },
};
```

## Error Handling

When middleware throws, the handler is not called:

```typescript
const requireAuth = zo.message.use(({ ctx }) => {
  if (!ctx.userId) {
    throw new Error("Unauthorized"); // Handler won't run
  }
  return {};
});
```

Errors are logged to the console:

```
SERVER: Middleware error for message "api.action": Unauthorized
```

## Middleware Order

Middleware runs in the order it's composed:

```typescript
const first = zo.message.use(() => {
  console.log("First");
  return { step: 1 };
});

const second = first.use(({ ctx }) => {
  console.log("Second, step is", ctx.step);
  return { step: 2 };
});

const third = second.use(({ ctx }) => {
  console.log("Third, step is", ctx.step);
  return { step: 3 };
});

// Logs: First, Second (step 1), Third (step 2)
// Handler receives ctx.step = 3
```

## Best Practices

### 1. Keep Middleware Focused

```typescript
// ✅ Good: Single responsibility
const requireAuth = zo.message.use(({ ctx }) => {
  if (!ctx.userId) throw new Error("Unauthorized");
  return {};
});

// ❌ Bad: Does too much
const doEverything = zo.message.use(async ({ ctx, payload }) => {
  validateAuth(ctx);
  validatePermissions(ctx);
  logRequest(payload);
  checkRateLimit(ctx);
  updateMetrics(ctx);
  return {};
});
```

### 2. Reuse Middleware

Create a library of reusable middleware:

```typescript
// middleware/auth.ts
export const requireAuth = zo.message.use(({ ctx }) => {
  if (!ctx.userId) throw new Error("Unauthorized");
  return {};
});

export const requireAdmin = requireAuth.use(({ ctx }) => {
  if (ctx.role !== "admin") throw new Error("Forbidden");
  return { isAdmin: true };
});

// middleware/logging.ts
export const withLogging = zo.message.use(({ ctx, payload }) => {
  console.log(`[${ctx.userId}]`, payload);
  return {};
});
```

### 3. Type Safety

Middleware return types are inferred:

```typescript
const withUser = zo.message.use(async ({ ctx }) => {
  const user = await database.getUser(ctx.userId);
  return {
    user: {
      name: user.name,
      email: user.email,
    },
  };
});

// Handler knows about ctx.user
const appRouter = zo.router(router, {
  api: {
    action: ({ ctx }) => {
      ctx.user.name; // string - fully typed!
      ctx.user.email; // string - fully typed!
    },
  },
});
```

### 4. Early Returns

Fail fast in middleware:

```typescript
const validate = zo.message.use(({ ctx, payload }) => {
  if (!ctx.userId) {
    throw new Error("No user ID");
  }

  if (!payload.valid) {
    throw new Error("Invalid payload");
  }

  return { validated: true };
});
```

## Common Patterns

### Authentication

```typescript
const authenticated = zo.message.use(({ ctx }) => {
  if (!ctx.token) throw new Error("No token");
  return { authenticated: true };
});
```

### Authorization

```typescript
const authorized = authenticated.use(({ ctx }) => {
  if (!ctx.permissions.includes("write")) {
    throw new Error("No permission");
  }
  return { authorized: true };
});
```

### Timing

```typescript
const timed = zo.message.use(() => {
  const start = Date.now();
  return {
    startTime: start,
    getElapsed: () => Date.now() - start,
  };
});
```

### Caching

```typescript
const cache = new Map();

const cached = zo.message.use(async ({ payload }) => {
  const key = JSON.stringify(payload);

  if (cache.has(key)) {
    return {
      cached: true,
      data: cache.get(key),
    };
  }

  return { cached: false };
});
```
