# React Integration

Zocket provides first-class React support with type-safe hooks for building real-time applications.

## Installation

```bash
bun add @zocket/react
```

The React package has peer dependencies on `react` (18+) and `@zocket/core`.

## Quick Example

```tsx
import { ZocketProvider, useZocket } from "@zocket/react";
import type { AppRouter } from "./server";

function App() {
  return (
    <ZocketProvider<AppRouter>
      url="ws://localhost:3000"
      headers={{ username: "Alice" }}
    >
      <Chat />
    </ZocketProvider>
  );
}

function Chat() {
  const { client, useEvent } = useZocket<AppRouter>();
  const [messages, setMessages] = useState([]);

  useEvent(client.on.chat.onMessage, (data) => {
    setMessages((prev) => [...prev, data]);
  });

  const sendMessage = (text: string) => {
    client.send.chat.message({ text });
  };

  return (
    <div>
      {messages.map((msg) => (
        <div key={msg.id}>
          {msg.user}: {msg.text}
        </div>
      ))}
      <button onClick={() => sendMessage("Hello!")}>Send</button>
    </div>
  );
}
```

## Features

### Type-Safe Hooks

All hooks are fully typed based on your router:

```tsx
useEvent(client.on.chat.onMessage, (data) => {
  data.user; // string - fully typed!
  data.text; // string - fully typed!
});
```

### Automatic Cleanup

Event listeners are automatically cleaned up when components unmount.

### Provider Pattern

Single WebSocket connection shared across your app.

## Architecture

```
<ZocketProvider>
    ↓
  Creates WebSocket connection
    ↓
  Provides client to children
    ↓
<YourComponents>
    ↓
  Use useZocket() and useEvent()
```

## Best Practices

### 1. Single Provider

Wrap your app once at the top level:

```tsx
// ✅ Good: Single provider at root
function App() {
  return (
    <ZocketProvider url="ws://localhost:3000">
      <Router />
    </ZocketProvider>
  );
}

// ❌ Bad: Multiple providers
function Component() {
  return (
    <ZocketProvider url="ws://localhost:3000">
      <Child />
    </ZocketProvider>
  );
}
```

### 2. Type Safety

Always pass the router type:

```tsx
// ✅ Good: Typed
<ZocketProvider<AppRouter> url="ws://localhost:3000">

// ❌ Bad: Untyped
<ZocketProvider url="ws://localhost:3000">
```

### 3. Error Boundaries

Wrap with error boundaries:

```tsx
<ErrorBoundary>
  <ZocketProvider<AppRouter> url="ws://localhost:3000">
    <App />
  </ZocketProvider>
</ErrorBoundary>
```
